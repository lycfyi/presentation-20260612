第 1–9 条
便携桥梁、攻城器械、火炮、战船、地下通道…… 九条全在讲军事工程。
第 10 条

“In painting, I can do everything possible.”

我们把他记成「画家」，但他把自己写成 一个能在工程、水利、解剖、绘画之间自由穿行、把一件事从头做成的人。

Source: Google Arts & Culture · Britannica
重点不是「他多才多艺」这种鸡汤，而是：文艺复兴稀缺的从来不是「最会画的」或「最会造炮的」， 而是「能跨领域架桥、拿一个问题从头做到尾」的人。这正是 AI 时代重新变稀缺的能力。
01 · 为什么是「文艺复兴」

为什么现在是新的「文艺复兴」时期

工具变革 · 新工具重塑认知与创作
当年铅笔、透视法（消失点）、显微镜、望远镜、印刷术 —— 人类第一次能更精准地记录、观察、传播知识。
今天AI 开发工具、云基础设施、新 IDE 与 agent 工作流，正像当年的印刷术，把「构建复杂系统的门槛」压到前所未有的低。
领域不再彼此孤立 · 级联效应
太空、AI、机器人 —— 多个黄金时代同时到来，一个领域的进步在加速另一个。领域之间没有清晰边界，进步呈现「正反馈 + 级联效应」。
Source: Werner Vogels · AWS re:Invent 2025 · “Echoes of Evolution and Innovation”
这页回答一个问题：Vogels 凭什么说我们在进入一次「文艺复兴」，而不是随便拿个词蹭热度。两个最直观的结构性信号： ① 工具变革：当年铅笔、透视法、显微镜、望远镜、印刷术让人第一次能精准地记录、观察、传播知识；今天 AI 开发工具、云、IDE 与 agent 工作流，正像印刷术一样把「构建复杂系统」的门槛压到极低。 ② 领域不再孤立：太空、AI、机器人多个黄金时代同时到来，一个领域的突破在加速另一个，边界消失，呈现正反馈 + 级联效应。 所以他把「AI 会取代我吗」这个问题，直接换成了「我会不会进化」。下一页用一张对照图，把达·芬奇和今天的 Builder 摆在一起。

02
去「Title（头衔）」化

头衔不是天经地义的，它是分工的副产品。

先别急着讲「去 title」。先问一个更根本的问题：title 到底是从哪来的？ 它不是自古就有的，它是「分工」这件事发展到一定程度的必然产物。 我们用一条时间轴，把分工史的拐点摆出来——你会发现 title 的激增和塌缩，都跟着分工层级走。

工业革命把 title 越分越细，
AI 正在把它收回去

过去 250 年，工作越切越细，每切一刀，就多一个 title。AI 是历史上第一次让这条曲线掉头向下。

Source: Adam Smith · 制针厂分工 · Where Did Job Titles Come From · The Great Flattening
1776 斯密制针厂：把造针拆成 18 道工序，单人产能从 ~20 枚/天飙到 ~4800 枚/天——分工是生产力的起点，但这时还没有 title。 1900s 工业革命 + 泰勒制：上百号人在大工厂干专门活，旧行会管不过来，于是直接借军队层级——中尉变经理、上尉变总监、上校变 VP。title 第一次激增。 1960s 起知识工作兴起：一个 VP 可能谁都不管，「senior 某某」和普通版没差别——title 与实际工作脱钩，开始通胀。 今天：Gartner 预测 2026 年 20% 的组织用 AI 砍掉过半中层（大扁平化）。分工层级被抽掉，title 跟着失重。 一句话：title 是分工的副产品，分工层级越多头衔越多；AI 抽掉中间层，头衔自然塌缩。 【分工的上行段——以前：把流程切得越细，title 就长得越多】 每一次"把一件事拆成更多步骤"，就诞生一批新岗位、新头衔，这是工业化效率提升的核心机制。 1 → 18：亚当·斯密的制针厂，一根针被拆成 18 道独立工序，分给 18 类人（Panmure House · Econlib）。 规划 / 执行：泰勒制（科学管理）把"动脑的人"和"动手的人"彻底分成两类岗位。 324 → ~1000：美国官方职业数从 1850 年的 324 个，涨到今天约 1000 个；今天 60% 的工作在 1940 年不存在（IPUMS 1850 · MIT / Autor 2024）。 这就是 title 的第一层来源：流程被切碎以后，需要名字来标记每一段。 title 并不是天然存在，而是分工越来越细之后，用来命名每一段流程的标签。
02 · 分工反转

现在：AI 用端到端闭环，把“胶水岗位”收回系统

汇报、中转、协调这些“分工产生的胶水岗位”，正是 AI 最先吃掉的部分。中层被重新定义为“可被自动化的人肉中间件”。

汇报状态、进度、阻塞点，过去靠层层转述。
中转信息从一个岗位流向另一个岗位，需要人工接力。
协调拆任务、催进度、对齐上下文，靠会议和 manager 兜底。
→
端到端闭环执行、反馈、学习、改进在系统里连续发生。
Agent 编排任务路由和跨工具触发由 agent 接走。
人只审关键点人的位置上移到定义意图、做判断、为结果负责。
Source: Axios · Insights@Questrom
工业化用“越分越细”制造 title；AI 用“端到端闭环”把它们收回去。 这里重点不是岗位名消失，而是原来靠人做胶水的环节变成系统能力。
02 · 层级反转

层级曾是生物跨度逼出来的，AI 第一次把它压平

SPAN OF CONTROL

一个人只能有效带 3–8 人——写在生物性里的硬上限。人一多，信息就要有人转述、路由，于是长出中层，科层制是这条生物跨度的产物。而 AI 把汇报、跟踪、协调直接自动化，一个 manager 第一次能管几十人，组织自然少几层。

谁在扁平化 · 2023→2026
  • Meta：效率之年，“Flatter is faster”。
  • Amazon：一线贡献者 : 经理比例提高约 15%。
  • Google：砍掉 VP 与经理岗。
  • Microsoft / Intel / Bayer：压平管理层级、削掉数千名经理。

这就是去 title 化的第二层来源：层级减少以后，用来标记层级的 title 也跟着消失。

Source: Business Insider · Axios · HR Bench / Gallup
把层级的根挖到底：科层制不是管理偏好，是人类生物跨度的产物。一个人只能有效带 3–8 人，超过就需要有人转述、路由信息——中层就是这么长出来的，结构上和罗马军团（8 人配一个 decanus）一脉相承。两千年来层数可变，但「能做信息路由的只有人」这个前提没变。AI 第一次让这个前提松动：系统持续汇报、跟踪、协调，一个 manager 就能覆盖更多人，组织少几层也能运行——层级减少，title 跟着失重。
02 · 层级终局

3–8 人的上限，是人的上限，不是协调本身的上限。AI 是历史上第一个不受这条生物跨度限制的协调媒介——「信息路由」第一次可以不再由人来做。

不是用 AI 辅助管理，
而是用 AI 取代管理。 Block · 把公司重做成一个 Intelligence Layer

当公司变成一个 Intelligence Layer，中层不是变少——是彻底消失。

这页是「协助 → 取代」的递进收口，语气切到 dark 是刻意的。上一页讲的「AI 协助 manager 管几十人」是当下现实，但那还是把 AI 当成更好的对讲机。真正的拐点在于：3–8 人的跨度限制是人的硬上限，AI 不受它约束——它是历史上第一个能做信息路由、却不是人的协调媒介。 Block 的赌注因此很直接：不是用 AI 辅助管理，而是用 AI 取代管理；当公司本身变成一个 Intelligence Layer，中层不是被压平，是从结构里消失。停一拍再翻页。

不是「上几个 AI 工具」。是把公司本身改造成三件事：能被查询、动作成闭环、能力会叠加。 接下来一页一张图，逐个讲。
03 · 第一件事

让公司可被查询 make it queryable

真实案例 · QUERY
问 Lucius（AI 客服） 社区现在讨论什么需求比较多，然后原文复制让 Claude Code 查数据库 社区里的需求动态由 agent 汇总；现有字段（schema）能不能覆盖这些需求，一句话查出来。社区对话 × 数据库——公司里隔最远的两摊知识，一次问通。
举我自己最近一件事：我先问 Lucius——我的一个 agent——「社区里大家最近在要什么、有什么需求动态」，它把散在社区里的讨论汇总给我；紧接着我让 Claude Code 去查我自己的数据库，现有字段能不能接住这些需求。 你看这中间发生了什么：社区对话和数据库 schema——公司里隔得最远的两摊知识——我一次提问就把它们串起来了。 大多数公司做不到，知识散落在文档、代码、对话、指标里，没法被一次问到。 让公司 queryable，就是让任何人/agent 一次提问，就能把散落各处的知识即时调出、还能追问。这是后面闭环的前提。
03 · 第二件事

把动作连成闭环 close the loop

开环：执行完就没下文，复盘靠开会。闭环：结果立刻回流，系统自我修正。

真实案例 · TUTTI 自动审核
审核 prompt 先上线 ＋ 给用户留申诉口子 申诉一回流就成闭环（像推荐算法）：据此迭代 prompt，并实测 false positive / false negative rate，一轮轮收窄。
真实案例 · 让用户骂你
Claude Code 埋 regex 检测「wtf」「piece of shit」 静默记 is_negative:true。愤怒是最诚实的信号——早上线、早被骂、早知道真问题。
开环：做决策、执行，然后……没有然后了。复盘靠开会，反馈靠下季度，每个环节都在丢信息。 闭环：执行→反馈→学习→改进，每个动作结果立刻回流，系统自我修正。 拿我们 Tutti 的自动审核做例子：内容审核、账号审核这种事，怎么写出一个又准又稳的 prompt，其实挺头疼的。但你不用一上来就追求完美——你只要把 prompt 上线，让它去跑真实的内容/账号审核，同时给用户留一个「申诉 / 抱怨」的口子。用户一旦来申诉，这条反馈就回流了，闭环就成了，有点像推荐算法：你拿用户反馈反过来迭代 prompt，还能实测它的 false positive / false negative rate，一轮轮收窄。 再讲个更狠的：「让用户骂你」其实是个严肃的产品建议。Anthropic 在 Claude Code 里埋了 regex，检测「wtf」「piece of shit」这类词，静默记录 is_negative:true——愤怒是最诚实的信号。调查问卷里没人说真话，但没人会对着一个让他们抓狂的产品客气。所以早点上线、早点被骂、早点知道真正的问题在哪——这也是一条闭环。 公司的速度不等于执行速度，等于「环路转一圈」的速度。中央那个旋转光点就是这个意思。
03 · 第三件事

增加被动技能 (各种 Buff)

每加一个能力（自动化 / agent / 记忆 / 闭环），公司基线永久抬高一截，叠加生效。

真实案例 · 主动 vs 被动
Anker 录音豆 / Plaud 要你主动点，Limitless 永远被动监听 同样是录音：一个得你想起来按一下，忘了就丢；一个挂在背景里永远在听。被动技能的价值，正在于「不用你每次操心」。
真实案例 · BUFF
Hermes 常驻读 Railway / Sentry 日志 → 自动开 PR 碰到重复类型的 error （需要叠加在记忆能力），直接开 PR。这层能力一旦装上，就永久在背景里替我盯线上、自己修——这就是一个被动技能。
被动技能 = 不用你每次操心、自动在背景里生效的能力。自动化、agent、记忆、闭环—— 每加一个，公司的基线就被永久抬高一截，而且叠加生效（右上那条上扬虚线）。 举个我自己的：我让 Hermes（一个 bot）常驻读 Railway 和 Sentry 的监控日志，碰到 error log 不是发个告警，而是直接开一个 PR——这层能力一旦装上，就永久在背景里替我盯线上、自己修，这就是一个被动技能。 再举个消费品上的对照：Anker 的录音豆、Plaude 这类录音设备，都得你「想起来主动点一下」才开始录，一忘就把信息漏掉了；而 Limitless 那个挂脖子的吊坠是永远被动监听——它不需要你每次操心，背景里一直在听、一直在记。这就是「主动」和「被动」的根本差别：被动技能的价值，恰恰在于它不占用你的注意力、永远在线生效。 这就是 AI Native 公司的复利：今天的投入变成明天的默认能力。
